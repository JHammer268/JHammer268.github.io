Heirarchical Clustering:

  • Lets generate some data to start:
set.seed(1234)
par(mar=c(0,0,0,0))
x<- rnorm(12, mean=rep(1:3, each=4), sd=0.2)
y <- rnorm(12, mean=rep(c(1,2,1), each=4), sd=.2)

plot(x,y, col="blue", pch=19, cex=2)
text(x+ 0.05, y+0.05, labels=as.character(1:12))

Observation:

There are three pretty obvious groups of data on this plot, as one would expect based on the parameters passed to rnorm above. The different specifications for the mean arg for x and y generate the clusters around 1,2,3 on the x axis, and 1,2,1 on the y-axis. So let’s Look at a few differnet ways of clustering and analyizing this example dataset:

hclust()

  • Next, load the data into a dataFrame, and break it up intp trees:
df <- data.frame(x=x,y=y)
distxy <- dist(df)

hClustering <- hclust(distxy)

par(mar=c(1,3,2,1))
plot(hClustering)

Observation:

  • Note that the Dendrogram does not specify how many clusters there are. It shows all possible clustering levels, so the zero value on the y axis would give you the trivial answer where every datapoint is it’s own cluster, whereas higher up on the grap at y=2 the dataset has been lumped into two groups. While this dataset has 3 pretty obvious clusters, in real data it is not always intuituve where the correct level of clustering falls. A geologist would refer to this as an example of “Lumping vs Splitting.”

  • Merging Methods: This refers to the moethod used to calculate the distance betwen points/clusters. Two common methods are Complete and Average to determine size/location of newly merged points/clusters. This can have a large effect on the outcome of the clster analysis.

for (i in 1:40) {

PCA - find which variables independedntly affect the data vs which are correlative with each other to reduce the number of variables needed

  • SVD Singular Value Decomposition: find a smaller Ranked Matirx that still accuarately predicts outcomes. SVD is a matrix decomposition: \[ X = UDV^T \] Where the columns of U are orthognonal (left singular vectors), the columns of V are orthognoal (right singular vectors) and D is a diagonal Matrix(singular values). This means that U corresponds to rows of X, and V corresponds to columns of X. Therefore:

  • PCA Principal components are equal to the right singular values if you first scale(subtact the mean and divide by standard deviation) the variables.

Components of SVD: U and V

svd1 <- svd(scale(dmOrdered))

par(mfrow=c(1,3), mar=c(4,4,4,4))

image(t(dmOrdered)[ ,nrow(dmOrdered):1])
plot(svd1$u[ ,1], 40:1, xlab="Row", ylab="First Left Singular Vector", pch=19)
plot(svd1$v[ ,1], xlab="Column", ylab="First Right Singular Vector", pch=19)

Components of SVD: Variance explained: D

  • D is the diagonal Matrix component.

  • The values of D represent the “magnitute of effect” or strength for each variable (column).

  • the svd function orders the diagonal by strength.

  • In the plots below, the left plot shows the raw D values, while the right is normalized. So the right plot shows that the first column is responsible for 40% of effect. Column2 approximately 15%, etc.

#SVD Component D
par(mfrow=c(1,2))
plot(svd1$d, xlab = "Column", ylab="Singular Value", pch=19, col="blue")
plot(svd1$d^2/sum(svd1$d^2), xlab="Column", ylab="Singular Value", pch = 19, col="purple")
