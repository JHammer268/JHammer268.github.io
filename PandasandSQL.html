Now I'll look at a more complicated query involving a subquery. Let's find out how many unique customers in the database have purchased a Jazz track. SQL First:¶

In [250]:
QUERY = ''' SELECT count(ID) from(SELECT Invoice.CustomerId as ID from Invoice
            JOIN InvoiceLine ON InvoiceLine.InvoiceId = Invoice.InvoiceID
            JOIN Track ON Track.TrackID = InvoiceLine.TrackID
            JOIN Genre ON Genre.GenreID = Track.GenreID
            WHERE Genre.Name = "Jazz"
            GROUP BY Invoice.CustomerID) as sub;
         '''
c.execute(QUERY)
#df = pd.DataFrame(c.fetchall())
c.fetchall()

QUERY = ''' SELECT Genre.name, count(Genre.name) as num FROM Genre
            JOIN Track ON Genre.GenreID = Track.GenreID
            WHERE Track.Milliseconds < ?
            GROUP BY Genre.Name
            ORDER BY num DESC
            Limit 10;
         '''
c.execute(QUERY, (average,))  #Note that the QUERY wants my variable tuple form.

In [271]:
QUERY = ''' SELECT Genre.name, count(Genre.name) as num FROM Genre, 
                (SELECT Avg(Milliseconds) as average FROM Track) AS sub
            JOIN Track ON Genre.GenreID = Track.GenreID
            WHERE Track.Milliseconds < average
            GROUP BY Genre.Name
            ORDER BY num DESC
            Limit 10;
         '''
c.execute(QUERY)

Conclusion:¶

I put this notebook together to illustrate some of the similarities and differences between Pandas filtering and aggregation and SQL Queries, as well as to illustrate a simple interaction between SQL and Python. One of the main take away points to me is how much a SQL database can help streamline an analysis by querying out the pertinent information. This can help make your code simpler and more readable. Also, performing Database queries can be much faster and less memory intense compared to the equivalent operations in Pandas. Conversely, once the data has been brought into a DataFrame all of the analysis and visualization tools of Python become available. Thanks for reading!
